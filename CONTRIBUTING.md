# Contributing to Techno Support VMS

## Quick Standards
1. **Branching:** Use `feat/name` or `fix/name`.
2. **Commits:** Use [Conventional Commits](docs/standards/git-workflow.md).
3. **Tests:** Must pass locally. Run `./scripts/test_all.sh`.
4. **Coverage:** Must not drop below 80%.

## Documentation
- Read [Go Standards](docs/standards/go.md)
- Read [C++ Standards](docs/standards/cpp.md)
- Read [Rust Standards](docs/standards/rust.md)

## Code of Conduct
Be respectful and constructive in code reviews.
